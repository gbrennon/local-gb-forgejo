## 2. bootstrap.sh — Current Expected Version

### Goal
`bootstrap.sh` must be the latest version: it sources `.env`, detects the container runtime by querying the daemon (not the CLI), tears down pods cleanly before recreating them (podman), creates the Forgejo admin user, fetches a real runner token from the Forgejo API, and registers the runner idempotently.

### Current State
The `bootstrap.sh` in the repo is an **older version**. It does not source `.env`, does not create the admin user, and generates a random hex secret instead of fetching a real token from Forgejo's API. It must be replaced.

### Correct `bootstrap.sh`

Replace the file with exactly the following content:

```bash
#!/usr/bin/env bash
set -euo pipefail

SECRET_FILE="forgejo_secret.txt"
RUNNER_CONFIG="/data/.runner"
FORGEJO_HOST_PORT="1234"

# ---------------------------------------------------------------------------
# Load .env file
# ---------------------------------------------------------------------------
ENV_FILE=".env"
if [[ -f "$ENV_FILE" ]]; then
  set -a
  source "$ENV_FILE"
  set +a
else
  echo "ERROR: .env file not found. Copy .env.example to .env and fill in values." >&2
  exit 1
fi

# ---------------------------------------------------------------------------
# Detect the actual container runtime by checking which daemon is reachable.
# CLI availability is irrelevant — what matters is which daemon owns the socket.
# ---------------------------------------------------------------------------
if podman info &>/dev/null 2>&1; then
  COMPOSE="podman-compose"
  CONTAINER_RUNTIME="podman"
elif docker info &>/dev/null 2>&1; then
  COMPOSE="docker compose"
  CONTAINER_RUNTIME="docker"
else
  echo "ERROR: neither podman nor docker daemon is reachable." >&2
  exit 1
fi
echo "Detected container runtime: $CONTAINER_RUNTIME"

# For runner registration: podman-compose run is outside the pod network,
# so we reach Forgejo via the host. Docker shares the compose network.
if [[ "$CONTAINER_RUNTIME" == "podman" ]]; then
  FORGEJO_INTERNAL_URL="http://host.containers.internal:${FORGEJO_HOST_PORT}"
else
  FORGEJO_INTERNAL_URL="http://forgejo:3000"
fi

FORGEJO_HOST_URL="http://localhost:${FORGEJO_HOST_PORT}"

# ---------------------------------------------------------------------------
# Wrapper: podman-compose creates pods — 'down' tears down the pod cleanly.
# docker compose up -d is idempotent natively.
# ---------------------------------------------------------------------------
compose_up() {
  if [[ "$CONTAINER_RUNTIME" == "podman" ]]; then
    $COMPOSE down 2>/dev/null || true
    $COMPOSE up --detach "$@"
  else
    $COMPOSE up -d "$@"
  fi
}

# Start core services
compose_up forgejo forgejo-db

echo "Waiting for Forgejo to be healthy..."
until $CONTAINER_RUNTIME inspect --format='{{.State.Health.Status}}' forgejo 2>/dev/null | grep -q "healthy"; do
  sleep 3
  echo "  still waiting..."
done
echo "Forgejo is ready."

# ---------------------------------------------------------------------------
# Idempotent: skip registration if runner config already exists in the volume
# ---------------------------------------------------------------------------
if $CONTAINER_RUNTIME run --rm \
    -v forgejo_runner_data:/data \
    busybox test -f "$RUNNER_CONFIG" 2>/dev/null; then
  echo "Runner already registered, skipping."
else
  # Ensure the Forgejo admin user exists (no-op if already created)
  echo "Ensuring Forgejo admin user exists..."
  $CONTAINER_RUNTIME exec forgejo \
    forgejo admin user create \
    --username admin \
    --password "${FORGEJO_ADMIN_PASSWORD}" \
    --email admin@local.dev \
    --admin \
    2>&1 | grep -v "already exists" || true

  # Fetch a runner registration token from the Forgejo API
  echo "Fetching runner registration token from Forgejo API..."
  forgejo_secret=$(curl -sf \
    -X POST \
    -u "admin:${FORGEJO_ADMIN_PASSWORD}" \
    "${FORGEJO_HOST_URL}/api/v1/admin/runners/registration-token" \
    | grep -o '"token":"[^"]*"' \
    | cut -d'"' -f4)

  if [[ -z "$forgejo_secret" ]]; then
    echo "ERROR: failed to retrieve runner registration token." >&2
    exit 1
  fi

  echo "$forgejo_secret" > "$SECRET_FILE"
  echo "Token saved to $SECRET_FILE"

  $COMPOSE run --rm forgejo-runner \
    forgejo-runner create-runner-file \
    --connect \
    --instance "$FORGEJO_INTERNAL_URL" \
    --name local-runner \
    --secret "$forgejo_secret"
fi

compose_up forgejo-runner
echo "All services are up."
```

### Verification
```bash
chmod +x bootstrap.sh
./bootstrap.sh
# Expected final line: "All services are up."
```

After running, verify the runner is registered:
```bash
curl -s -u "admin:${FORGEJO_ADMIN_PASSWORD}" \
  http://localhost:1234/api/v1/admin/runners \
  | grep -o '"name":"[^"]*"'
# Expected: "name":"local-runner"
```

### Goal
The runner must accept jobs tagged `runs-on: ubuntu-latest` so that GitHub Actions workflows run without modification.

### Current State

### Actions

```bash

  forgejo-runner create-runner-file \
  --connect \
  --instance "$FORGEJO_INTERNAL_URL" \
  --name local-runner \
  --secret "$forgejo_secret" \
```

```bash
  | grep -o '"labels":\[[^]]*\]'
```

## 4. Repository Mirroring

### Goal
Mirror repositories from GitHub, GitLab, and Bitbucket into Forgejo so CI can run locally.

### Architecture
```
GitHub / GitLab / Bitbucket
        │  push event (webhook) or scheduled poll
        ▼
   Forgejo (local, port 1234)
        │  detects new commits
        ▼
  forgejo-runner → executes workflow jobs
```

### 4.1 Mirror a Repository via API (scriptable, preferred)

```bash
curl -s -X POST \
  -u "admin:${FORGEJO_ADMIN_PASSWORD}" \
  -H "Content-Type: application/json" \
  http://localhost:1234/api/v1/repos/migrate \
  -d '{
    "clone_addr": "https://github.com/YOUR_ORG/YOUR_REPO.git",
    "auth_token": "YOUR_GITHUB_PAT",
    "mirror": true,
    "mirror_interval": "8h",
    "repo_name": "YOUR_REPO",
    "repo_owner": "admin",
    "service": "github",
    "private": false
  }'
```

For GitLab, set `"service": "gitlab"` and use a Project Access Token (`read_repository` scope).
For Bitbucket, set `"service": "bitbucket"` and use an App Password (`Repositories: Read`).
For any other Git host, set `"service": "git"` and omit `auth_token` (use `auth_username`/`auth_password` instead).

### 4.2 Mirror via UI (alternative)

`http://localhost:1234 → + → New Migration → [GitHub / GitLab / Bitbucket / Git]`

### 4.3 Webhook-triggered Sync (near real-time)

Polling (`mirror_interval`) introduces delay. For instant sync on push, configure a webhook on the upstream platform.

**Prerequisites:** Forgejo must be reachable from the internet. Use a tunnel:
```bash
cloudflared tunnel --url http://localhost:1234
# or
ngrok http 1234
```

**Generate a Forgejo API token** (do not use the admin password for webhooks):
```
http://localhost:1234/user/settings/applications → Generate Token
Scope: repository
```

**Webhook payload URL:**
```
POST https://YOUR_PUBLIC_URL/api/v1/repos/{owner}/{repo}/mirror-sync
Header: Authorization: token YOUR_FORGEJO_API_TOKEN
```

Configure this URL as a push webhook on GitHub (`Settings → Webhooks`), GitLab (`Settings → Webhooks`), or Bitbucket (`Settings → Webhooks`).

### Verification
```bash
# Trigger a manual sync
curl -s -X POST \
  -u "admin:${FORGEJO_ADMIN_PASSWORD}" \
  http://localhost:1234/api/v1/repos/admin/YOUR_REPO/mirror-sync
# Then check the repo in the UI — last sync time should update
```

## 5. CI Workflows

### 5.1 GitHub Actions — No Changes Required

Forgejo reads `.github/workflows/` automatically. A mirrored GitHub repo with existing workflows will trigger them on sync without any modification, provided the runner label matches (`ubuntu-latest`).

### 5.2 Forgejo-specific Override

To run a different pipeline on Forgejo without touching the GitHub workflow, create `.forgejo/workflows/ci.yml` in the repo. Forgejo checks this path first and uses it instead of `.github/workflows/`.

```yaml
# .forgejo/workflows/ci.yml
on: [push, pull_request]

jobs:
  build:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - name: Build
        run: make build
```

### 5.3 GitLab CI / Bitbucket Pipelines — Rewrite Required

`.gitlab-ci.yml` and `bitbucket-pipelines.yml` are **not** understood by Forgejo. For repos mirrored from these platforms, a `.forgejo/workflows/ci.yml` must be written that replicates the intent of the original pipeline using GitHub Actions syntax.

Mapping reference:
| Original concept | Forgejo Actions equivalent |
|---|---|
| GitLab `stage` | `jobs.<job_id>` with `needs:` for ordering |
| GitLab `script` | `steps[].run` |
| Bitbucket `step` | `jobs.<job_id>.steps` |
| `$CI_*` variables | `${{ github.* }}` context variables |

### 5.4 Secrets

Secrets are not mirrored from upstream platforms. Add them manually per repo:

```bash
curl -s -X PUT \
  -u "admin:${FORGEJO_ADMIN_PASSWORD}" \
  -H "Content-Type: application/json" \
  http://localhost:1234/api/v1/repos/admin/YOUR_REPO/actions/secrets/MY_SECRET \
  -d '{"data": "secret_value"}'
```

Or via UI: `Repo → Settings → Secrets and Variables → Actions → New Secret`

## 6. API Quick Reference

All endpoints are relative to `http://localhost:1234/api/v1`. Auth: `-u "admin:${FORGEJO_ADMIN_PASSWORD}"`.

| Action | Method | Path |
|---|---|---|
| List repos | GET | `/repos/search` |
| Create mirror | POST | `/repos/migrate` |
| Trigger mirror sync | POST | `/repos/{owner}/{repo}/mirror-sync` |
| List runners | GET | `/admin/runners` |
| Get runner token | POST | `/admin/runners/registration-token` |
| Add secret | PUT | `/repos/{owner}/{repo}/actions/secrets/{name}` |
| List workflows | GET | `/repos/{owner}/{repo}/actions/workflows` |

Interactive API docs: `http://localhost:1234/api/swagger`
